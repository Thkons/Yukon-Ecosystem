--------------------------------------------
1. ΠΕΡΙΓΡΑΦΗ ΕΡΓΟΥ
--------------------------------------------

Η εφαρμογή HOS Hospital App, αποτελεί μία νοσοκομειακή εφαρμογή σχεδιασμένη για τη διαχείριση διαφρόρων λειτουργιών του οικοσυστήματος νοσοκομείων Yukon. Υποστηρίζει πολλαπλούς ρόλους χρηστών, όπως:

- Ασθενείς  
- Γιατρούς  
- Γραμματείς  
- Τεχνική Υποστήριξη   

Η εφαρμογή επιτρέπει υποβολή αιτήματος για προγραμματισμό ραντεβού, επικοινωνία μεταξύ ασθενών και γιατρών, μεταφόρτωση αρχείων, καθώς και εσωτερική διαχείριση του συστήματος.

--------------------------------------------
2. ΡΟΛΟΙ ΧΡΗΣΤΩΝ & ΛΕΙΤΟΥΡΓΙΕΣ
--------------------------------------------

[1] ΑΣΘΕΝΗΣ  
- Δημιουργία και σύνδεση σε λογαριασμό  
- Διαχείριση προσωπικών στοιχείων
- Μεταφόρτωση ιατρικών αρχείων (π.χ. εξετάσεις, συνταγές)  
- Αίτηση για ραντεβού / ακύρωση αίτησης 
- Προβολή ιστορικού αρχείων & ραντεβού  

[2] ΓΙΑΤΡΟΣ  
- Υποβολή αιτήματος για δημιουργία λογαριασμού
- Διαχείριση προσωπικών στοιχείων
- Πρόσβαση σε φακέλους ασθενών  
- Προβολή μεταφορτωμένων εγγράφων  
- Έγκριση ραντεβού / ακύρωση έγκρισης
- Καταγραφή διαγνώσεων και θεραπευτικών σχεδίων  

[3] ΓΡΑΜΜΑΤΕΑΣ  
- Επεξεργασία ραντεβού και χρηστών 
- Έγκριση δημιουργίας λογαριασμού για γιατρούς/γραμματείς/μέλος τεχνικής υποστήριξης 
- Διαχείριση ημερολογίου ραντεβού  

[4] ΤΕΧΝΙΚΗ ΥΠΟΣΤΗΡΙΞΗ  
- Παρακολούθηση της κατάστασης του οικοσυστήματος  
- Αντιμετώπιση προβλημάτων και συνδέσεων 
- Δημιουργία alerts

3. ΑΠΑΙΤΗΣΕΙΣ ΣΥΣΤΗΜΑΤΟΣ
--------------------------------------------

- Java JDK 17 ή νεότερο  
- JavaFX SDK 17+  
- MariaDB ή MySQL Server  

--------------------------------------------
4. ΕΓΚΑΤΑΣΤΑΣΗ & ΡΥΘΜΙΣΗ
--------------------------------------------

1ο ΒΗΜΑ: Βάση Δεδομένων  
- Ανοίξτε τον MariaDB/MySQL Client 
- Δημιουργήστε βάση δεδομένων `test` 
- Κάντε Import το αρχείο `test.sql` από τον φάκελο `db/`  
- Ανοίξτε το XAMPP Control Panel
- Κάντε Start τον Apache και τον MySQL Server

2ο ΒΗΜΑ: Εκτέλεση  
- Μεταφορά φακέλου στο htdocs του xampp 
- Εκτελέστε το `HOS.java` από το IDE  
ή  
- Εκτελέστε το αρχείο JAR από το φάκελο `dist/`:
  HOS.jar

--------------------------------------------
5. ΣΥΝΟΨΗ ΒΑΣΗΣ ΔΕΔΟΜΕΝΩΝ
--------------------------------------------

Κύριοι Πίνακες:
- `users`: στοιχεία σύνδεσης και ρόλοι χρηστών  
- `applicants`: υποψήφιοι χρήστες
- `appointments`: ραντεβού ασθενών  
- `available_dates`: ημερομηνίες για ραντεβού  
- `available_times`: ώρες για ραντεβού   
- `doctors`: στοιχεία γιατρών  
- `patients`: στοιχεία ασθενών  
- `patient_files`: μεταφορτωμένα αρχεία  
- `reports`: δελτία σφαλμάτων  

--------------------------------------------
6. ΣΗΜΕΙΑ ΑΣΦΑΛΕΙΑΣ
--------------------------------------------
 
- Έλεγχος εγκυρότητας εισόδων χρηστών πριν από εισαγωγή στη βάση  
- Τα αρχεία αποθηκεύονται με μοναδικά ονόματα  
- Για όλες τις κύριες λειτουργίες απαιτείτε έγκριση 
- Ο ρόλος της γραμματείας - admin έχει πλήρη εξουσία – χρήση με προσοχή
